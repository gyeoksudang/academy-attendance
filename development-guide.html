📊 구글 시트를 데이터베이스로 활용하는 장점

✅ 관리자 친화적

  • 직관적 인터페이스: 엑셀과 유사한 친숙한 환경
  • 실시간 편집: 웹에서 바로 데이터 수정 가능
  • 권한 관리: 사용자별 접근 권한 세밀 제어

🔄 개발자 친화적

  • Apps Script 연동: JavaScript로 쉬운 데이터 조작
  • REST API: 웹 애플리케이션과 간단한 연동
  • 실시간 동기화: 데이터 변경 즉시 반영
💡 핵심: 복잡한 데이터베이스 설정 없이도 강력한 데이터 관리 시스템 구축 가능

🚨 주요 기술적 문제들

❌ CORS 문제

증상:

  • "Access-Control-Allow-Origin" 오류
  • Preflight 요청 차단
  • fetch 요청 실패

원인:

  • 불필요한 headers 설정
  • 복잡한 fetch 옵션

✅ 해결책

단순한 요청 사용:

// ❌ 문제 코드 fetch(url, { method: 'GET', headers: {'Content-Type': 'application/json'} }); // ✅ 해결 코드 fetch(url);

❌ QR 코드 생성 실패

증상:

  • "QRCode is not defined"
  • 라이브러리 로딩 실패
  • QR 이미지 생성 안됨

✅ 해결책

안정적인 라이브러리 사용:

// qrcodejs 라이브러리 사용 new QRCode(element, { text: data, width: 256, height: 256, correctLevel: QRCode.CorrectLevel.L });

🤖 LLM에게 요구해야 할 핵심 사항

🎯문제 상황을 구체적으로 명시

좋은 예시:

"GitHub Pages에서 Apps Script로 fetch 요청시 'Access-Control-Allow-Origin' 오류가 발생합니다. Content-Type 헤더 때문에 preflight 요청이 발생하는 것 같습니다."

🔧해결 방법을 직접적으로 요청

좋은 예시:

"fetch 요청에서 모든 headers와 옵션을 제거해서 단순한 GET 요청으로 만드는 방법을 알려주세요."

📝실제 작동하는 완전한 코드 요청

좋은 예시:

"QRCode is not defined 오류를 해결하고, 실제로 QR 코드가 생성되는 완전한 HTML 파일을 만들어주세요. qrcodejs 라이브러리를 사용해서 안정적으로 작동하도록 해주세요."
6/8

📋 구체적인 요청 전략

🎯검증된 해결책 명시

"CORS 문제를 해결하기 위해 fetch 요청에서 mode, cache, headers 등 모든 옵션을 제거하고 가장 단순한 fetch(url) 형태만 사용해주세요."

📚라이브러리 선택 기준 제시

"QR 코드 생성을 위해 qrcodejs 라이브러리 (https://cdnjs.cloudflare.com/ajax/libs/qrcodejs/1.0.0/qrcode.min.js) 를 사용하고, CorrectLevel.L 옵션으로 안정성을 높여주세요."

🔍단계별 테스트 방법 요청

"Apps Script에서 testFunctions()를 실행해서 데이터가 정상적으로 조회되는지 확인하는 방법과, 브라우저 콘솔에서 오류를 확인하는 방법을 포함해주세요."

📦복사-붙여넣기 가능한 형태 요청

"구글 시트에 데이터를 한 번에 입력할 수 있도록 탭으로 구분된 텍스트 블록 형태로 제공해주세요."

📝 효과적인 프롬프트 구성

3️⃣ 완전한 구현 요청
